Why your model has 12 attention heads

Your pipeline uses the pretrained encoder all-MiniLM-L6-v2. That model was designed and trained with a hidden size of 384 and 12 heads, so each head gets an equal 32-number slice.

Simple idea

Think of attention heads like 12 small reading teams looking at the same sentence at the same time. Each team looks for different relationships, such as:

  • which words belong together,
  • which number matches which label,
  • which part of a sentence explains another part.
Why 12?
The model stores each token as 384 values.
It splits those values evenly across 12 heads.
384 / 12 = 32

Why not 16 or 20?
  • 20 does not fit cleanly: 384 / 20 = 19.2.
  • 16 would fit mathematically: 384 / 16 = 24.
  • To keep 32 values per head with 16 heads, the model would need 512 total values.
  • So 16 would be a different model design, not this MiniLM checkpoint.
  • The head count is fixed during pretraining, not chosen later by the pipeline.

Non-technical takeaway: the model uses several smaller “views” in parallel instead of one giant view. That helps it notice different kinds of patterns without making the model too heavy.
